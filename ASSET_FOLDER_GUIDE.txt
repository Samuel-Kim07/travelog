===========================
Travelog Asset Folder Guide
===========================

Media storage folders for the Travelog prototype:

assets/icons/Audio/
  - Generated/recorded guide audio files are saved here directly.
  - Use playable compressed source formats such as .webm, .ogg, .m4a, .mp3.
  - The app no longer saves these as ZIP files, so users can open/listen to them directly.
  - Each upload also creates a small .json metadata file next to the media file.

assets/icons/Video/
  - Generated/selected guide video files are saved here directly.
  - Use playable compressed source formats such as .mp4 or .webm.
  - The app no longer saves these as ZIP files, so users can open/watch them directly.
  - Each upload also creates a small .json metadata file next to the media file.

Important
---------
For prototype testing, a GitHub token is stored only in the browser localStorage.
Do not commit your token into GitHub files.
For production, replace direct browser token upload with a server-side or GitHub App flow.


MINHO TEST MEDIA WIRED IN APP
- Minho intro video: assets/icons/Video/t_log_video.mp4
- Minho guide audio: assets/icons/Audio/Test_log.m4a

Use the GitHub Pages relative path in the app, not the github.com/blob page URL.
The blob page is for viewing files in GitHub; the relative path is what audio/video tags can play.
